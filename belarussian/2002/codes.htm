The pupil of 11th form, Petya Bulochkin is fond of coding of the information. Once, when he ate the next candie on breakfast and thought about his favourite theme, he had a fascinating idea: «It is possible to code the information so, that nobody, even me, will not decode it». Petya didn't know the advantages of such way of coding, but the idea was very interesting . There was a science conference in gimnasium, where Petya studied, and he decided to participate in it.

«I invented an essentially new way of coding, which is impossible to decode , – Petya said. – At the present time my coding works only for numbers. Let's consider an integer N, having t digits (in decimal notation ). Deleting the digits from this number by all possible ways, we will get some new numbers. Some way of deleting is to delete nothing. If we delete all digits, 0 is left. Let's find the sum of all this numbers. This sum is the code of N».

There were loud applauds. Then there were questions to Petya, which he successfully answered. But one notice confused him: : «If we realize your algorithm on the computer, than even for number from 100 digits, it will work very long. Your life is not enough to code only one number, having 100 digits. Your algorithm can't be applied on practice ». It was necessary to answer something. «No, no, and no. Tomorrow I'll bring the realization of my algorithm, that will code the numbers, having 100 digits, in no more than 1 second», - that was the answer of Petya.

Of course, Petya couldn't write such a program. So, you should write it. Before this, let's consider Petya's method of coding on an example. Let's N = 109. After deleting we get the numbers:

Task. Your program should input the integer N (1<=N <=10100 ) and determine the integer S – the code of N (using Petya's method of coding).

Input. Input consists of one line, which contains one integer N. The left digit of N is not zero.

Output. You should output an integer S – the code of N. You should not output the leading zeroes.
